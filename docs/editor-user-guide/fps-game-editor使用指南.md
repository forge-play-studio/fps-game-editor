# fps-game-editor使用指南

> 这是 `docs/editor-user-guide/fps-game-editor使用指南.html` 的 Markdown 发布副本，方便上传到其他文档平台。维护时以 `fps-game-editor使用指南.html` 为准；修改用户指南后，再同步更新本文件。

给关卡设计师、玩法设计师、技术美术和项目接入开发者使用的教学型说明。按学习路径进入，完成第一个可保存修改，再用手册、快捷键和排障内容巩固工作流。

## 文档定位

这套文档面向使用 `fps-game-editor` 的外部读者。它解释用户任务和编辑器概念，而不是复制源码实现细节。

> **阅读方式：**第一次使用时先按“新手学习路径”走一遍；已经熟悉界面时，可以直接查工作区、快捷键或排障章节。

### 任务教程

## 新手学习路径

这条路径帮助第一次使用编辑器的读者理解工作区，并完成一次从进入编辑模式、创建或选择节点、修改属性、保存场景到回到游戏预览的基础闭环。

### 完成后你应该能够

1. 说清楚 Scene View、层级、检查器、资产和历史面板分别负责什么。
2. 在示例项目中进入编辑器并找到可操作对象。
3. 使用检查器或 Transform 工具做一次可保存修改。
4. 判断保存后 authoring source 和 runtime scene 的关系。

### 推荐顺序

1. 先读工作区总览，理解每个面板和基本概念。
2. 再按第一个教程完成一次带检查点的操作。
3. 操作过程中需要快捷键时，查看本页的快捷键和鼠标操作章节。
4. 如果保存、选择或资源创建没有达到预期，查常见问题排障。
5. 完成一次保存闭环后，再回看工作区总览，补齐选择、资产和保存边界。

> 完成检查：你应该知道 `Q/W/E/R` 对应选择、移动、旋转、缩放工具，也知道 `Cmd/Ctrl + S` 会触发项目 adapter 的保存链路。

## 编辑器工作区总览

编辑器工作区是用户进行场景 authoring 的主要界面。公共编辑器负责选择、工具切换、Transform gizmo、Inspector host、历史状态和通用命令；项目 adapter 负责把这些命令落到项目自己的 schema、资源系统、runtime binding 和保存链路。

工作区布局：
- 层级 / Hierarchy
- Scene View / 选择、相机、Transform Gizmo
- 检查器 / Inspector
- 资产 / 历史 / Assets / History

### 什么时候使用

1. 创建或调整场景节点。
2. 放置资源并修改 Transform。
3. 查看和编辑选中节点的 Inspector 属性。
4. 保存 authoring source，并让项目生成 runtime scene。
5. 排查选择、资源、保存和运行态投影是否一致。

### 界面区域

1. `层级`：显示场景节点树，支持选择、重命名、分组和上下文操作。
2. `Scene View`：显示可编辑场景，承载选择框、相机导航和 Transform gizmo。
3. `检查器`：显示当前选择的属性，支持 document-backed 字段编辑。
4. `资产`：浏览项目资产，项目 adapter 可以把资源创建为场景节点。
5. `历史`：查看来自 EditorSession 的文档操作历史。

> 在 `mini-game-lab` 示例中，本地保存会写 `src/config/editor-scene.json`，并编译 `src/config/scene.json`。其他项目可以使用不同路径和保存宿主。

## 创建第一个可保存场景修改

完成一次最小编辑闭环：进入编辑器，创建或选择一个节点，修改它的位置，保存场景，然后回到游戏运行态检查结果。

> **示例入口：**在 `mini-game-lab` 中，`Enter Editor`、`Save Scene` 和 `Discard & Run Game` 是本地开发入口。正式项目可以由平台按钮或自己的宿主流程进入编辑器。

### 1. 进入编辑器

1. 打开项目页面。
2. 点击页面顶部的 `Enter Editor`。
3. 等待中央 Scene View、左侧层级、右侧检查器和底部资产面板出现。

为什么这样做：进入编辑模式后，当前 GameWorld 会切换到编辑器的 EditorWorld，用户动作会进入编辑器会话，而不是直接驱动游戏玩法。

> 检查点：你应该看到完整编辑器工作区。

### 2. 创建或选择一个节点

1. 如果层级里已经有节点，点击一个可选中节点。
2. 如果需要新节点，切到底部 `资产` 标签，搜索或找到一个资源，然后点击资源创建 GameObject。
3. 也可以在层级面板中使用 `+ Group` 创建一个分组节点。

为什么这样做：编辑器所有后续操作都围绕“当前选择”进行。

> 检查点：被选中的节点应该在层级中高亮，右侧检查器会显示这个节点的属性。

### 3. 修改 Transform

1. 按 `W` 切到移动工具，或在 Scene View 工具条选择移动。
2. 拖拽移动手柄，让节点在场景中移动一小段距离。
3. 也可以在检查器里修改 Transform 数值。

为什么这样做：Transform 是最基础的可保存编辑，可以验证选择、Inspector、gizmo、document patch 和 runtime projection 的完整链路。

> 检查点：节点在 Scene View 中的位置应该变化，检查器里的 Transform 数值也应该反映这次变化。

### 4. 保存并回到游戏运行态

1. 点击 `Save Scene`，或使用 `Cmd/Ctrl + S`。
2. 保存后切回 Game Mode 或使用项目提供的运行入口。
3. 如果要放弃本次编辑，在 `mini-game-lab` 中点击 `Discard & Run Game`。

为什么这样做：公共编辑器会把编辑意图交给项目 adapter，项目 adapter 保存 authoring source，并编译运行时需要的 scene 数据。

> 检查点：游戏运行态重新出现后，你应该能看到刚才保存的节点变化。

## 快捷键和鼠标操作

本节把当前编辑器的工具切换、Scene View 导航、选择、文档操作、层级菜单和面板操作放在同一个用户指南里。单独打开本页或导出的 Markdown 时，也能看到完整参考。

### 最小记忆集合

### 工具切换

`Q/W/E/R` 对应选择、移动、旋转、缩放，是最常用的编辑节奏。

### 视图导航

中键平移，滚轮推进/拉远，右键拖拽进入飞行视角，`WASD/QE` 在飞行视角中移动。

### 文档操作

`Cmd/Ctrl + S` 保存，`Cmd/Ctrl + Z` 撤销，删除和复制行为由项目 adapter 决定是否可用。

### 工具与 Transform

工具快捷键只切换当前编辑工具，不会直接修改场景数据。真正的 Transform 变化来自 gizmo 拖拽或 Inspector 数值编辑。

| 按键或入口 | 作用 | 使用场景 |
| --- | --- | --- |
| `Q` | 切换到选择工具。 | 普通选择、框选前的基础模式。 |
| `W` | 切换到移动工具。 | 显示移动 gizmo，可沿轴向、平面或视图平面移动。 |
| `E` | 切换到旋转工具。 | 显示旋转 gizmo，绕 X/Y/Z 单轴旋转。 |
| `R` | 切换到缩放工具。 | 显示缩放 gizmo，可单轴缩放或等比缩放。 |
| 世界 / 本地 | 在 Scene View 工具条切换 Transform 坐标空间。 | 需要按世界方向或对象自身方向移动、旋转、缩放时使用。 |
| 移动手柄 | 轴向箭头、平面方块和中心自由手柄同时可用。 | 精确移动时用轴向或平面；快速拖动时用中心自由手柄。 |
| 吸附 | 在 Scene View 工具条切换移动、旋转、缩放步进吸附。 | 需要按固定网格、角度或比例编辑时使用。 |
| 放置 | 在 Scene View 工具条切换关闭、XZ 地面或场景表面放置。 | 把对象放到地面或已有表面上时使用。 |
| 对齐 / 分布 | 多选后从 Scene View 工具条执行位置对齐或分布。 | 需要快速整理一组选中对象的位置时使用。 |

### Scene View 导航

Scene View 相机操作只改变观察视角，不写入 authoring source。飞行视角中，滚轮用于调整飞行速度。

| 按键或鼠标 | 作用 | 说明 |
| --- | --- | --- |
| `F` | 聚焦当前选中的 GameObject。 | 需要先选中一个可聚焦对象。 |
| `Esc` | 取消当前 gizmo 拖拽或活动操作。 | 不等同于撤销历史。 |
| 滚轮 | 推进/拉远摄像机。 | 右键飞行模式下改为调整飞行速度。 |
| 中键拖拽 | 平移 Scene View。 | 适合微调视图构图。 |
| `Alt` + 左键拖拽 | 环绕 Scene View。 | 围绕当前视图目标旋转观察。 |
| `Alt` + 中键拖拽 | 平移 Scene View。 | 和中键拖拽类似。 |
| `Alt` + 右键拖拽 | 推进/拉远 Scene View。 | 适合没有滚轮或需要连续 dolly 时使用。 |
| 右键拖拽 | 进入飞行视角并转动相机。 | 按住右键时可使用 `W/A/S/D/Q/E` 移动。 |
| 右键 + `W`/`S` | 飞行视角前进 / 后退。 | 只移动相机，不移动对象。 |
| 右键 + `A`/`D` | 飞行视角左移 / 右移。 | 按相机朝向的左右方向移动。 |
| 右键 + `Q`/`E` | 飞行视角下降 / 上升。 | 用于快速调整观察高度。 |

### 鼠标操作

| 操作 | 作用 | 说明 |
| --- | --- | --- |
| 左键 | 选择对象。 | 在空白区域拖拽可以框选。 |
| `Shift` + 左键 | 追加选择对象。 | 不会清空已有选择。 |
| `Cmd/Ctrl` + 左键 | 切换对象的选中状态。 | 已选中的对象会被移出选择，未选中的对象会被加入选择。 |
| 双击对象 | 选择并聚焦对象。 | 适合快速定位场景中的对象。 |
| `Shift` + 拖拽 | 框选时追加对象。 | 用于扩大当前选择集。 |
| `Cmd/Ctrl` + 拖拽 | 框选时切换对象选中状态。 | 用于从选择集中加入或移除一批对象。 |

### 选择

Scene View 和层级树都会驱动同一组选择状态。正常情况下，层级高亮、Scene View 选择目标和 Inspector 摘要应该指向同一对象。

| 入口 | 作用 | 说明 |
| --- | --- | --- |
| Scene View 点击 | 选择被点击的 GameObject。 | 对象需要可选中，并且项目 adapter 需要提供稳定 runtime binding。 |
| 层级树点击 | 从 Hierarchy 中选择 GameObject。 | 适合 Scene View 中对象重叠或难以点中时使用。 |
| 层级树 `Shift` / `Cmd/Ctrl` | 追加或切换选择。 | 和 Scene View 多选语义一致。 |
| 右键层级节点 | 打开 GameObject 操作菜单。 | 可执行聚焦、重命名、创建子分组、分组选中对象、删除等操作。 |

### 文档操作

保存、撤销、重做、删除和复制都依赖项目 adapter 或接入宿主提供对应语义。快捷键触发后，如果项目拒绝操作，编辑器应该保持当前数据不被破坏。

| 快捷键 | 作用 | 说明 |
| --- | --- | --- |
| `Cmd/Ctrl + S` | 保存场景。 | 保存 editor-scene，并由项目 adapter 编译或导出运行时 scene 数据。 |
| `Cmd/Ctrl + Z` | 撤销。 | 撤销上一步文档操作，前提是项目提供可撤销 patch。 |
| `Cmd/Ctrl + Shift + Z` | 重做。 | 恢复刚撤销的文档操作。 |
| `Ctrl + Y` | 重做。 | 兼容 Windows 风格重做快捷键。 |
| `Delete` / `Backspace` | 删除当前选中的 GameObject。 | 受 protected、locked、read-only 和项目删除规则限制。 |
| `Cmd/Ctrl + D` | 复制当前选中对象。 | 需要项目接入 Duplicate 规则；如果菜单或操作置灰，说明当前项目未启用。 |

### 层级

| 操作 | 作用 | 说明 |
| --- | --- | --- |
| 右键节点 | 打开节点上下文菜单。 | 可聚焦、重命名、创建子分组、分组选中对象或删除。 |
| `F` / Focus in Preview | 聚焦当前节点。 | 节点需要可选中、未锁定且非 protected。 |
| Rename | 开始重命名节点。 | read-only、locked 或 protected 节点不可重命名。 |
| 重命名时 `Enter` | 提交重命名。 | 输入框失焦时也会提交。 |
| 重命名时 `Esc` | 取消本次重命名。 | 保留原名称。 |
| Add Empty Group | 创建空分组。 | 在节点菜单中创建子分组；在空白区域菜单中创建顶层分组。 |
| Group Selection | 把当前可移动的多选节点编入新分组。 | 要求项目已连接分组 pipeline。 |
| `Delete` | 删除选中节点。 | 只删除允许删除的选中节点。 |

### 面板

| 入口 | 作用 | 说明 |
| --- | --- | --- |
| Assets 标签 | 搜索项目资产，点击资产创建 GameObject。 | 资产列表和创建规则由项目 adapter 提供。 |
| History 标签 | 查看来自 EditorSession 的文档操作历史。 | 用于确认撤销/重做链路。 |
| Inspector 输入框 | 编辑选中对象的序列化属性。 | Transform 数值通常支持直接编辑。 |
| 多选 Inspector | 批量编辑多个选中 GameObject 的共享 Transform 数值。 | 只应编辑所有选中对象共享且可持久化的字段。 |

### 限制与注意

1. 快捷键需要编辑器或 Scene View 获得输入焦点；焦点在浏览器地址栏、外部宿主 UI 或文本输入框时，快捷键不会触发场景编辑。
2. `Q/W/E/R` 在普通编辑状态下切换工具；右键飞行视角中，`WASD/QE` 用于移动相机。
3. 删除、复制、资产创建、Inspector 字段写入和保存都可能被项目 adapter 拒绝。
4. 层级菜单中的 Copy、Paste、Copy Transform、Paste Transform、Locked、Do Not Serialize 等占位项如果置灰，表示当前版本或当前项目尚未启用。
5. Scene View 相机移动、平移、环绕和 dolly 只改变观察视角，不会写入场景数据。
6. 当前版本记录默认快捷键；如果你的项目提供自定义绑定界面，以项目说明为准。

### 快捷键不生效

> **先看焦点：**大部分快捷键失效都和输入焦点有关。先点击 Scene View 或编辑器空白处，再重试快捷键。

1. 如果 `Q/W/E/R` 不切换工具，确认没有正在编辑 Inspector 输入框或层级重命名输入框。
2. 如果 `Cmd/Ctrl + S` 没有保存，确认当前处于 Editor Mode，且项目没有处于 busy 状态。
3. 如果 `Delete` 没有删除，检查选中节点是否 protected、locked、read-only，或者项目是否拒绝删除。
4. 如果右键飞行视角无法移动，确认右键仍按住，并且 Scene View 处于活动状态。
5. 如果复制、分组或资产创建没有效果，检查当前项目是否提供了对应 adapter 能力。

## 常见问题排障

### 你可能看到

1. 点击 `Enter Editor` 后没有出现编辑器工作区。
2. Scene View 中无法选中对象，或层级选中和视图选中不一致。
3. 点击资产后没有创建节点。
4. Inspector 字段显示了，但输入后没有保存。
5. `Save Scene` 或 `Cmd/Ctrl + S` 后，游戏运行态没有变化。

### 检查顺序

1. 确认你真的进入了编辑模式。正常时应该看到层级、Scene View、检查器和底部 dock。
2. 在层级中点击节点。正常时节点应该高亮，检查器应该显示当前选择。
3. 在 Scene View 中点击同一个对象。正常时选择状态应该和层级同步。
4. 打开底部 `资产` 标签。正常时应该能看到项目 adapter 暴露的资源数量和筛选框。
5. 修改一个基础 Transform 字段或拖动移动 gizmo。正常时 Scene View 和检查器数值应该同步变化。
6. 执行保存。正常时项目 adapter 应该保存 authoring source，并生成或导出 runtime scene。

### 工作区没有出现

确认使用了项目提供的编辑入口，例如 `mini-game-lab` 中的 `Enter Editor`，并检查控制台是否有 bridge、Inspector loader 或资源加载错误。

### 无法选择或 Inspector 为空

优先从层级面板选择节点，检查项目 adapter 是否为 runtime 对象建立了持久化 binding。

### 保存后运行态没有变化

确认保存动作已经触发。在 `mini-game-lab` 中检查 `editor-scene.json` 和 `scene.json` 是否都发生预期变化。
