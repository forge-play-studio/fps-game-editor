原则

  1. 用户指南只放外部读者需要阅读的内容。
  2. fps-game-editor使用指南.html 是用户教学入口的权威版本；fps-game-editor使用指南.md 只是发布副本。
  3. fps-game-editor使用指南.html 和导出的 fps-game-editor使用指南.md 必须自包含，单独带到其他电脑或平台时不能缺少关键用户内容。
  4. 可独立查阅的用户参考页可以拆成单独 HTML，但不能成为用户指南入口或 Markdown 发布副本的必需内容来源。
  5. Agent 参考只放写作流程、事实来源和质量检查。
  6. Manifest、稳定术语和模板只放在系统文档中。
  7. 页面先从用户任务解释编辑器使用，再补充实现边界。
  8. 项目特定行为必须明确标注。
  9. 稳定概念和术语应跨页面复用。

受众

受众需要什么
beginner第一次使用编辑器，需要安全引导路径。
designer编辑场景、对象、资源和属性的关卡或玩法设计师。
adapter_developer把编辑器接入项目 schema、runtime、assets 和保存流的开发者。
agent_or_author需要按固定教学契约维护和扩展用户指南。

内容类型

类型用途必需标题
learning-path把多个页面组织为学习路线。目标、适合谁、学习路线、完成检查

troubleshooting帮助读者诊断并恢复常见失败。现象、常见原因、检查顺序、修复动作、需要收集的信息

稳定术语

  • 编辑器：用户操作的 authoring environment。
  • 项目 adapter：把编辑器动作映射到项目 schema、资源、runtime binding 和保存链路的项目侧层。
  • authoring source：编辑器保存的项目编辑源数据。
  • runtime scene：游戏运行态消费的场景数据。
  • 节点：层级中的可选中场景对象。
  • 资源：被场景引用的模型、图片、音频等资源。
  • Inspector：查看和编辑选中节点属性的面板。

文档模板

模板只供生成新章节时参考，不展示在用户指南里。

教程模板

  1. 目标：说明读者会完成什么，以及完成后的可观察结果。
  2. 适合谁：说明目标读者和默认前置知识。
  3. 开始前：列出入口、项目、场景、资源和限制。
  4. 步骤：每个关键动作包含操作、原因和检查点。
  5. 常见问题：按现象、原因、修复方式组织。
  6. 下一步：链接相关教程、手册或参考。

手册页模板

  1. 概念：说明概念、界面区域或工作流是什么。
  2. 什么时候使用：列出适合和不适合的使用场景。
  3. 界面或数据位置：说明 UI 区域、authoring source、runtime scene 或 adapter 位置。
  4. 工作方式：按用户动作、编辑器解释、adapter 写回、预览/保存说明流程。
  5. 限制与注意：列出项目特定差异和未稳定行为。
  6. 相关主题：链接相关页面。

参考页模板

  1. 范围：说明覆盖和不覆盖的命令、字段、路径、限制或错误。
  2. 条目：使用表格或可扫描列表，写清精确名称、路径、命令或快捷键。
  3. 默认行为：说明默认规则。
  4. 限制：说明例外、版本差异和项目特定边界。

排障页模板

  1. 现象：列出读者能看到的 UI、控制台或保存结果。
  2. 常见原因：列出最可能原因。
  3. 检查顺序：从低成本检查开始，每一步说明正常结果。
  4. 修复动作：按适用情况、操作、验证组织。
  5. 需要收集的信息：说明继续排查所需状态、配置、错误和复现步骤。
