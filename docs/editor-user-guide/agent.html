Agent Reference

Agent 参考文档

这页约束 agent 如何维护对外教学文档。它不面向最终用户，也不存放模板正文；Manifest 和系统模板在独立的系统文档中。

打开用户文档 查看系统模板

文档分层

  1. index.html 是外部教学文档，只放用户需要直接阅读的学习路径、教程、手册、参考和排障内容。
  2. index.md 是 index.html 的 Markdown 发布副本，用于上传其他平台，不作为源文档。
  3. shortcuts.html 是外部快捷键页面，按用户查阅方式分类整理快捷键与鼠标操作。
  4. agent.html 是 Agent 参考文档，只放写作目标、更新流程、事实来源和质量检查。
  5. system.html 是系统模板与 Manifest，只放内容契约、稳定术语、页面类型和模板骨架。
不要把 Manifest、系统模板、占位标题、内部写作约束或源码遍历计划写进 index.html。

更新流程

  1. 先读 index.html，理解当前用户能看到的教学路径。
  2. 再读 system.html，确认受众、内容类型、必需标题和稳定术语。
  3. 读取必要事实来源，确认当前代码、README、示例项目或架构说明是否已经变化。
  4. 只更新用户真正需要阅读的内容；内部判断、模板引用和 Manifest 变化写回本页或 system.html。
  5. 更新后检查导航、锚点、外部读者语气和是否混入内部文档生产资料，再同步 index.md。

教学写作规则

读者优先

  1. 第一次打开编辑器的人需要路径、目标、检查点和安全回退。
  2. 已经会基本操作的人需要更短的任务步骤和更清晰的边界说明。
  3. 项目接入开发者需要 adapter、authoring source、runtime scene、保存链路等概念，但仍应从任务出发。
  4. 排障读者需要现象、原因、定位入口和修复动作。

写作要求

  1. 先告诉读者会完成什么，再解释内部实现。
  2. 每个关键步骤后给出检查点：应该看到什么、保存后应该发生什么、失败时先看哪里。
  3. 用编辑器概念组织内容，例如场景、节点、资源、属性、预览、保存、导出。
  4. 明确区分用户动作、编辑器行为和项目 adapter 的职责。
  5. 对尚未稳定或依赖具体项目的功能，用“当前示例项目中”或“项目 adapter 通常需要”标注。

避免

  1. 直接从源码文件顺序生成文档。
  2. 把内部设计计划写成用户承诺。
  3. 用“很简单”“显然”等词压低读者困惑。
  4. 在教程里塞完整 API 参考。
  5. 把一个页面同时写成教程、手册和排障表。

事实来源

  1. README.md：包安装、接入边界、发布策略和用户文档入口。
  2. agent.md：仓库协作规则、保存链路长期原则和文档分层规则。
  3. docs/*.md 和 docs/*.html：框架设计、迁移记录和内部实现说明。
  4. examples/mini-game-lab/README.md：当前真实示例项目的本地验证入口。
  5. 直接相关源码：只在需要确认实际 UI、快捷键、保存行为或 adapter 语义时读取。

检查清单

  1. 用户页导航只出现面向使用者的章节。
  2. 用户页没有 Manifest、模板占位符、Agent 写作规则或“生成文档”的指令。
  3. index.md 和 index.html 表达同一套用户入口内容，且 Markdown 顶部声明 HTML 为准。
  4. 快捷键页按工具、视图、鼠标、选择、文档、层级和面板分类，不把未实现能力写成已可用承诺。
  5. 教程保留目标、步骤、原因、检查点、常见问题和下一步。
  6. 手册页解释概念和边界，不承诺尚未稳定的项目行为。
  7. 参考和排障内容可扫描，且标注项目特定差异。
  8. 内部链接能跳转，文件列表保持 HTML-first：Markdown 只作为发布副本。
