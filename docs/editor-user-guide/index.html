文档定位

这套文档面向使用 fps-game-editor 的外部读者。它解释用户任务和编辑器概念，而不是复制源码实现细节。

阅读方式：第一次使用时先按“新手学习路径”走一遍；已经熟悉界面时，可以直接查工作区、快捷键或排障章节。

新手学习路径

这条路径帮助第一次使用编辑器的读者理解工作区，并完成一次从进入编辑模式、创建或选择节点、修改属性、保存场景到回到游戏预览的基础闭环。

完成后你应该能够

  1. 说清楚 Scene View、层级、检查器、资产和历史面板分别负责什么。
  2. 在示例项目中进入编辑器并找到可操作对象。
  3. 使用检查器或 Transform 工具做一次可保存修改。
  4. 判断保存后 authoring source 和 runtime scene 的关系。

推荐顺序

  1. 先读工作区总览，理解每个面板和基本概念。
  2. 再按第一个教程完成一次带检查点的操作。
  3. 操作过程中需要快捷键时，打开 快捷键和鼠标操作。
  4. 如果保存、选择或资源创建没有达到预期，查常见问题排障。
  5. 完成一次保存闭环后，再回看工作区总览，补齐选择、资产和保存边界。
完成检查：你应该知道 Q/W/E/R 对应选择、移动、旋转、缩放工具，也知道 Cmd/Ctrl + S 会触发项目 adapter 的保存链路。

编辑器工作区总览

编辑器工作区是用户进行场景 authoring 的主要界面。公共编辑器负责选择、工具切换、Transform gizmo、Inspector host、历史状态和通用命令；项目 adapter 负责把这些命令落到项目自己的 schema、资源系统、runtime binding 和保存链路。

什么时候使用

  1. 创建或调整场景节点。
  2. 放置资源并修改 Transform。
  3. 查看和编辑选中节点的 Inspector 属性。
  4. 保存 authoring source，并让项目生成 runtime scene。
  5. 排查选择、资源、保存和运行态投影是否一致。

界面区域

  1. 层级：显示场景节点树，支持选择、重命名、分组和上下文操作。
  2. Scene View：显示可编辑场景，承载选择框、相机导航和 Transform gizmo。
  3. 检查器：显示当前选择的属性，支持 document-backed 字段编辑。
  4. 资产：浏览项目资产，项目 adapter 可以把资源创建为场景节点。
  5. 历史：查看来自 EditorSession 的文档操作历史。
在 mini-game-lab 示例中，本地保存会写 src/config/editor-scene.json，并编译 src/config/scene.json。其他项目可以使用不同路径和保存宿主。

创建第一个可保存场景修改

完成一次最小编辑闭环：进入编辑器，创建或选择一个节点，修改它的位置，保存场景，然后回到游戏运行态检查结果。

示例入口：在 mini-game-lab 中，Enter Editor、Save Scene 和 Discard & Run Game 是本地开发入口。正式项目可以由平台按钮或自己的宿主流程进入编辑器。

1. 进入编辑器

  1. 打开项目页面。
  2. 点击页面顶部的 Enter Editor。
  3. 等待中央 Scene View、左侧层级、右侧检查器和底部资产面板出现。

为什么这样做：进入编辑模式后，当前 GameWorld 会切换到编辑器的 EditorWorld，用户动作会进入编辑器会话，而不是直接驱动游戏玩法。

检查点：你应该看到完整编辑器工作区。

2. 创建或选择一个节点

  1. 如果层级里已经有节点，点击一个可选中节点。
  2. 如果需要新节点，切到底部 资产 标签，搜索或找到一个资源，然后点击资源创建 GameObject。
  3. 也可以在层级面板中使用 + Group 创建一个分组节点。

为什么这样做：编辑器所有后续操作都围绕“当前选择”进行。

检查点：被选中的节点应该在层级中高亮，右侧检查器会显示这个节点的属性。

3. 修改 Transform

  1. 按 W 切到移动工具，或在 Scene View 工具条选择移动。
  2. 拖拽移动手柄，让节点在场景中移动一小段距离。
  3. 也可以在检查器里修改 Transform 数值。

为什么这样做：Transform 是最基础的可保存编辑，可以验证选择、Inspector、gizmo、document patch 和 runtime projection 的完整链路。

检查点：节点在 Scene View 中的位置应该变化，检查器里的 Transform 数值也应该反映这次变化。

4. 保存并回到游戏运行态

  1. 点击 Save Scene，或使用 Cmd/Ctrl + S。
  2. 保存后切回 Game Mode 或使用项目提供的运行入口。
  3. 如果要放弃本次编辑，在 mini-game-lab 中点击 Discard & Run Game。

为什么这样做：公共编辑器会把编辑意图交给项目 adapter，项目 adapter 保存 authoring source，并编译运行时需要的 scene 数据。

检查点：游戏运行态重新出现后，你应该能看到刚才保存的节点变化。

快捷键和鼠标操作

快捷键已经拆成独立页面，按工具、视图、鼠标、选择、文档、层级和面板分类。第一次使用编辑器时，可以先记住 Q/W/E/R、F、Esc、Cmd/Ctrl + S 和 Scene View 的鼠标导航。

工具切换

Q/W/E/R 对应选择、移动、旋转、缩放，是最常用的编辑节奏。

视图导航

中键平移，滚轮推进/拉远，右键拖拽进入飞行视角，WASD/QE 在飞行视角中移动。

文档操作

Cmd/Ctrl + S 保存，Cmd/Ctrl + Z 撤销，删除和复制行为由项目 adapter 决定是否可用。

常见问题排障

你可能看到

  1. 点击 Enter Editor 后没有出现编辑器工作区。
  2. Scene View 中无法选中对象，或层级选中和视图选中不一致。
  3. 点击资产后没有创建节点。
  4. Inspector 字段显示了，但输入后没有保存。
  5. Save Scene 或 Cmd/Ctrl + S 后，游戏运行态没有变化。

检查顺序

  1. 确认你真的进入了编辑模式。正常时应该看到层级、Scene View、检查器和底部 dock。
  2. 在层级中点击节点。正常时节点应该高亮，检查器应该显示当前选择。
  3. 在 Scene View 中点击同一个对象。正常时选择状态应该和层级同步。
  4. 打开底部 资产 标签。正常时应该能看到项目 adapter 暴露的资源数量和筛选框。
  5. 修改一个基础 Transform 字段或拖动移动 gizmo。正常时 Scene View 和检查器数值应该同步变化。
  6. 执行保存。正常时项目 adapter 应该保存 authoring source，并生成或导出 runtime scene。

工作区没有出现

确认使用了项目提供的编辑入口，例如 mini-game-lab 中的 Enter Editor，并检查控制台是否有 bridge、Inspector loader 或资源加载错误。

无法选择或 Inspector 为空

优先从层级面板选择节点，检查项目 adapter 是否为 runtime 对象建立了持久化 binding。

保存后运行态没有变化

确认保存动作已经触发。在 mini-game-lab 中检查 editor-scene.json 和 scene.json 是否都发生预期变化。
